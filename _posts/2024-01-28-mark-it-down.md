## 2. Text Styles {#head2}

### **bold:**  

>✅**Syntax**➡️(`** ** or __ __`)      
>✅**Example**➡️(`**This is bold** | __This also__`)      
>✅**Output**➡️(**This is bold** | __This also__)     


 ### _italic_:  

>✅**Syntax**➡️(`* * or _ _`)     
>✅**Example**➡️(`*this is italic* | _this also_`)     
>✅**Output**➡️(*this is italic* | _this also_)   


### **bold** and _italic_:  

>✅**Syntax**➡️(`*** *** or ___ ___`)    
>✅**Example**➡️(`***This is bold and italic*** | ___This also___`)    
>✅**Output**➡️(***This is bold and italic*** | ___This also___)    


### Bold and _nested italic_:  

>✅**Syntax**➡️(`**_ _**`)  
>✅ **Example**➡️(`**This is _italic_ inside bold**`)   
>✅**Output**➡️(**This is _italic_ inside bold**)   


### <sub>Subscript</sub>  

>✅**Syntax**➡️(`<sub> </sub>`)    
>✅**Example**➡️(`This is a <sub>SubScript</sub> text`)    
>✅**Output**➡️(This is a <sub>SubScript</sub> text)    


### <sup>Superscript</sup>  

>✅**Syntax**➡️(`<sup> </sup>`)    
>✅**Example**➡️(`This is a <sup>SuperScript</sup> text`)    
>✅**Output**➡️(This is a <sup>SuperScript</sup> text)


### <mark>Highlighting</mark>

>✅**Syntax**➡️(`<mark> </mark>`)    
>✅**Example**➡️(`This is a <mark>Highlighted text.</mark>`)    
>✅**Output**➡️(This is a <mark>Highlighted text.</mark>)   
>> The Markdown's own syntax is `==Highlighted text==`.
But The Markdown All in One(a VSCode extension) and my current Jekyll theme don't support it.     


### Strikethrough
>✅**Syntax**➡️(`~~  ~~`)    
>✅**Example**➡️(`~~Strikethrough~~`)    
>✅**Output**➡️(~~Strikethrough~~)  






## 3. <img  src="https://cdn-icons-png.flaticon.com/128/5771/5771269.png"  alt="paragraph"  width="50"  height="autho">Paragraphs {#head3}



✅**Syntax**  

> Use 2️⃣`SPACE`s(☄️☄️) at the end of a line to break it and start a new line.
> Press `ENTER` to create _blank lines_ in order to separate paragraphs or start a new paragraph(🛣️).


✅**Example**

> This is the first line of the first paragraph. ☄️☄️  
> This is the second line. ☄️☄️  
🛣️🛣️🛣️🛣️🛣️🛣️🛣️🛣️  
This is the first line of the second paragraph ☄️☄️  
> And this is the second line. ☄️☄️



> **Tip**:
* Keep lines left-aligned. 
* don't use `TAB` to indent paragraph. if you need, use  HTML entity for _non-breaking space_:
```  
   &nbsp;&nbsp;&nbsp;&nbsp;This is an indented Paragraph.   
```
* You can use HTML `<br>` element for line breaking.
{: .prompt-tip} 
  







## 4. <img  src="https://cdn-icons-png.flaticon.com/128/852/852907.png"  alt="blocs"  width="50"  height="auto">Blockquote {#head4}

### ❇️Turn a line or a paragraph to a blockquote.


✅**Syntax** ➡️ Put a Greater than sign ( > ) at the begining.   


✅**Example**⬇️
``` 
> I am turning this paragraph to a blocquote with just one _Greater-than sign (>).    
be aware that the lines mustn't have blank lines in between.  
The lines must be breaked by `SPACE` not separated by `ENTER`.  
```

✅**Output**⬇️ 

> I am turning this paragraph to a blockquote with just one _Greater-than sign (>).  
be aware that the lines mustn't have blank lines in between.  
The lines must be breaked by `SPACE` not separated by `ENTER`.



### ❇️Turn separated lines or more than one paragraph to blockquote.

✅**Syntax**:⬇️  
 Put a Greater-than sign( > ) at the begining and between each line.

✅**Example**:⬇️
```
> This is the first line of the first paragraph.  
Here is the second line.
Let's stop here.
>
> This is the Second paragraph.  
lets put them together in one Blockquote.  
```

✅**Output**⬇️

> This is the first line of the first paragraph.  
Here is the second line.
Let's stop here.
>
> This is the Second paragraph.  
lets put them together in one Blockquote.  



### ❇️Indented or nested blockquotes.

✅**Syntax**:➡️  Use Double(>>), triple(>>>) and more  Greater-than sign.

✅**Example**:⬇️
```
> This is outer blockquote.
>> This is inner blockquote.
> >> This line is the most inner one.
>> >> You can go deeper inside...
> >> >> deeper and deeper!
>> >> >> again
>> >> >>> agian and again
```

✅**Output**⬇️

> This is outer blockquote.
>> This is inner blockquote.
> >> This line is the most inner one.
>> >> You can go deeper inside...
> >> >> deeper and deeper!
>> >> >> again
>> >> >>> agian and again


### Inline Blockquote:

 ✅**Syntax**:➡️   `<blockquote></blockquote>`

 ✅**Example**:⬇️

## 6.Code In Markdown {#head6}

### Inline code


✅**Syntax**⬇️   
To provide inline code, wrap the code snippet in 2️⃣ backticks `` ` ` ``  

✅**Example**:⬇️  
```
Use the HTML element `<br>` instead of 2️⃣ space, to break a line.
```

✅**Output**➡️  
Use the HTML element `<br>` instead of 2️⃣ space, to break a line.  


### Code Blocks


✅**Syntax**⬇️  
To create Code Blocks, use 3️⃣ backticks(```) or tilde(~~~)before and after the code snippet, and provide the language name after initial backtics, in order to _Highlight the code syntax_.

✅**Example**:⬇️   
```
   ```html
     Code Block goes here...
     ```
```

✅**Output**⬇️  

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CodeChronicles</title>
</head>
<body>
    
</body>
</html>
```


> **Tip:** We can also use the TAB key to indent lines when creating code blocks, but putting triple backticks twice is easier than indenting each line.
{: .prompt-tip }





## 7.🔗Links: {#head7}
  
### Inline link:

✅**Syntax**  ➡️ `[wrap the __link text__ in brackets]( wrap the __link__ in parentheses "title goese here")`  

✅**Example** ➡️ `[Pirikli's _GitHub_](https://github.com/pirikli "Pirikli, a Poet and Programming Pilgrim")`  

✅**Output**➡️ [**Pirikli**'s _GitHub_](https://github.com/pirikli "Pirikli, a Poet and Programming Pilgrim")


### Reference link:

✅**Syntax**⬇️ 
```
[link text][refrence to another place inside doc]

[refrence to another place inside doc]: link-to-outside-websites "title"
```
The link title can be wraped in **" ", ' ' & ( )**.     

✅**Example** ⬇️
```
[My Linkedin] [ refrence-to-link ]

[ refrence-to-link ]: https://www.linkedin.com/in/pirikli/ "Pirikli"
```

✅**Output**⬇️  
[**_My Linkedin_**][refrence-to-link]

[refrence-to-link]: https://www.linkedin.com/in/pirikli/ "Pirikli"

### Direct Link:
✅**Syntax** ➡️ To quickly convert a URL or email address into a link, wrap it in double angel brackets.

✅**Example** ➡️ `<pirikliartan@gmail.com>`

✅**Output** ➡️ <pirikliartan@gmail.com>
  
> **Tip**: We add the `{:target="_blank"}` immediately after the link, in order to open the link in new tab:
```
[John Gruber on Markdown](https://daringfireball.net/projects/markdown/){:target="_blank"}
```
[John Gruber on Markdown](https://daringfireball.net/projects/markdown/){:target="_blank"}
{: .prompt-tip}

## 9. 📃Lists {#head9}

### Ordered list

✅**Syntax**:➡️ Use numbers adjacent to dots(1. 2. 3.)

✅**Example**:⬇️
```
1. This is the first item of this ordered list.
2. This is the second item.
3. The third item.
```

✅**Output**⬇️
1. This is the first item of this ordered list.
2. This is the second item.
3. The third item.

### Unordered list:

✅**Syntax**: ➡️ use asterisk * or dash -


✅**Example**:⬇️
```
* This is the first item of this unordered list.  
* This is the second one.
* and son on...
```

✅**Output**⬇️
* This is the first item of this unordered list.  
* This is the second one.
* and son on...


### Nested lists(indentation in lists):
✅**Syntax**: ➡️ Use `TAB` to indent nested lists

✅**Example**:⬇️
```
1. The first item with inner list:
    - first inner
    - second inner
2. The second item with nested list
    * first inner
    * second inner
        - one
        - two
            * A
            * B
```

✅**Output**⬇️
1. The first item with inner list:
    - first inner
    - second inner
2. The second item with nested list
    * first inner
    * second inner
        - one
        - two
            * A
            * B
  

### Task Lists:  

✅**Syntax**: ➡️ - [ ] task

✅**Example**:⬇️
```
- [ ] Wake Up
- [ ] Get Up
- [ ] Wash Up
- [ ] Rush Up
```

✅**Output**⬇️

- [ ] Wake Up
- [ ] Get Up
- [ ] Wash Up
- [x] Rush Up ❗

## 12. Heading IDs {#head12}

 I think about it as "Inner Link" or "IN Page Link".  
 I usually use it to create "Content Table" and "🦘Jump Up High".   
  
✅**Syntax**:⬇️

It's made up of two parts:

- **Custom ID:** This involves using a pound/hashtag sign, immediately followed by a custom ID, both enclosed in curly braces. Place this construction in front of the heading to which you want to refer.

- **Link to Custom ID:** For this, put the heading name inside square brackets right next to the pound sign, with the custom ID enclosed in parentheses.

✅**Example**:⬇️
```md
[Here is the Link to that Heading](#headid5)   

      
##### This is a heading we want to refer or link to {#headid5}
```

✅**Output**⬇️     

[Here is the Link to that Heading](#headid5)   
     
         

##### This is a heading we want to refer or link to {#headid5} 



## 13. Footnotes {#head13}

✅**Syntax**:⬇️    
* To make the first part of a Footnote (footnote refrence), we need 3 ingredients:     
    - 1️⃣suare brackets [ ], 2️⃣caret sign ^, 3️⃣and an identifier, number or text, with no space.      
* The second part (footnote itself) is made of the same set of square brackets, caret sign and identifiers + colon and text.
   
  
✅**Example**:⬇️
```
This is the refrence to premier footnote[^1]
Here is the refrence to second footnote[^second]

---
[^1]: Thi is First Footnote.
[^second]: Here is the scond Footnote.
```
✅**Output**⬇️

This is the refrence to the first footnote[^1].  
Here is the refrence to the second footnote[^second].


[^1]: **Thi is First Footnote.**
[^second]: **Here is the scond Footnote.**
