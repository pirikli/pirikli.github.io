During one of those courses, I stumbled upon what would become a lifetime ally. I was taking a course on Git when I became acquainted with the README file. I decided to write a clean README for my newly created GitHub repository. It was clean. It was elegant. Its syntax was simple and effective.

Here is what I discovered about Markdown:**#

For me, Markdown is a primitive markup language that, with its simple tools (#, *, !, =, -, ~), makes your life easy and fluent. I’ve always been jealous of life in primitive timesâ€”a simple and intentional life without the complications of today’s world. In the coding world, Markdown allows me to live such a simple and effective life. It allows me to document rapidly and deeply without engaging in the decorative syntax of HTML or CSS. I learn, I document, and I go.

1. Headers#

	##### Fifth Level Header
	###### Sixth Level Header

Alternate Syntax for H1 & H2: Use a horizontal line (ğŸŸ° & â–) under the heading text.

Note:

Initially, I was using headers in the wrong way. I was using them everywhere on the page without any order. Then I recognized that headers are meant to create hierarchy, not style. They are not decorative elements. They exist to establish structure. That is why they are ordinal. I realized that on a single page, you should use only one H1 heading.
